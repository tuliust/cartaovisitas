# 13 — Roadmap e itens futuros

## Standby

### Apple Wallet real

Status: standby.

Dependências:

- orçamento Apple Developer Program;
- conta Apple Developer;
- Pass Type ID;
- certificado `.p12`;
- WWDR;
- variáveis privadas na Vercel;
- ativação de flags;
- teste físico em iPhone.

### Google Wallet

Status: standby/futuro.

Dependências:

- Google Wallet Issuer Account;
- Google Cloud;
- Service Account;
- Issuer ID;
- Class ID;
- JWT assinado;
- aprovação/publicação.

## Futuro previsto

### Aprovação de cartões

Fluxo possível:

```text
rascunho → enviado para revisão → aprovado → publicado
```

### Página pública por área/equipe

Exemplos:

```text
/equipe/comunicacao
/equipe/diretoria
```

### Busca pública de contatos

Página para buscar contatos por nome, área ou cargo.

### Analytics avançado

- gráficos por período;
- ranking;
- dispositivos;
- origem/referrer;
- exportação CSV;
- dashboard executivo.

### Assinatura de e-mail

Gerar assinatura HTML/texto com dados do cartão e QR Code.

### FAQ/suporte

Página com instruções para colaboradores e admins.

### Modo evento

Cartões/QRs específicos para eventos, feiras e missões.

## Não necessário no momento

- Open Graph dinâmico por cartão.
- Duplicar cartão.
- Arquivar cartão.
- Exportação em lote de QR Codes.
- Alertas de cartões desatualizados.

## Documentação institucional final

Depois da estabilização técnica, criar manual institucional com:

- padrões de cargo em PT/ES/EN;
- padrão de departamento;
- padrão de telefone;
- uso de foto;
- política de atualização;
- governança de administradores;
- processo de solicitação de cartão.
